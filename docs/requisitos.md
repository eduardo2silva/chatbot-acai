# Requisitos do Sistema 
## 1. Visão Geral
Este documento descreve os requisitos do sistema de chatbot para pedidos em uma loja de açaí, desenvolvido utilizando n8n. 
## 2. Requisitos Funcionais (RF)
| Código | Descrição |
|---|-----------|
|	RF01 | O sistema deve apresentar um menu de opção de açaí. |
|	RF02 | O sistema deve permitir que o cliente escolha um produto. |
|	RF03 | O sistema deve solicitar o endereço de entrega. |
|	RF04 | O sistema deve permitir a escolha da forma de pagamento (PIX ou Cartão). |
|	RF05 | O sistema deve simular o pagamento conforme a opção escolhida. |
|	RF06 | O sistema deve registrar o pedido no banco de dados. |
|	RF07 | O sistema deve notificar o dono do estabelecimento com os dados do pedido. |
## 3. Requisitos Não Funcionais (RNF)
| Código | Descrição |
|---|-----------|
|	RNF01 | O sistema deve ser simples e intuitivo. |
|	RNF02 | O sistema deve funcionar em ambiente local. |
|	RNF03 | O banco de dados deve ser relacional (PostgreSQL). |
|	RNF04 | O chatbot deve responder em tempo aceitável ao usuário. |
